Getting Started

Interface

ZoneLocationPurpose
ViewportTopLive point cloud. 200,000 points at 60fps via Metal.
NodeviewBottom barDrag nodes from the palette. Wire them together.
CameraviewBottom barSliders, pads, and buttons for the selected node.

Navigating the Nodeview

1. Tap + to open the node palette organized by family.

2. Tap a node to add it to the canvas.

3. Drag from an output port (right) to an input port (left) to wire.

4. Tap any node to see its parameters in the Cameraview.

Navigating the Cameraview

Sliders adjust float parameters in real time — changes apply instantly to the GPU. Tap the expose icon on any slider to create an input port and wire a signal into it.

Port Types

Input vs Output Squares

SquarePositionMeaning
OutputRight side of nodeData this node produces. Drag FROM here. One output can feed many inputs.
InputLeft side of nodeData this node consumes. Drag TO here. Each input accepts exactly ONE wire. If unwired, uses default value.

FromToBehavior

The depth vs z distinction

OutputWhat It IsUse For
depthNearness 0-1: (far - raw_metres) / (far - near). 1 = at NEAR, 0 = at FAR. No scale or sign.Palette.t, Size, Shape-by-Depth, Gate thresholds — any effect that wants "how close."
zGeometry: (FOCUS - raw_metres) x SEPARATION. Signed, metrically scaled.Output.z pin placement, Lag/Trail/Spring/Velocity motion processing.

A point at 4m with FAR=2.5 reads depth=0 ("far, color it cold") but z=-7.5 view units (behind the wall). Wire depth into Output.z and the scene compresses flat. Wire z into Palette and colors are meaningless. Rule: depth feeds effects, z builds the cloud.

Your First Patch

1. Add Depth (Source). 2. Add Pinout (Grid). 3. Add Palette (Color). 4. Wire together:
